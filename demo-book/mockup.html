Demo Book with RevealJS Format Support

This mockup demonstrates the new RevealJS format support for Quarto books. Notice the easel icons (🎨) in the sidebar next to each chapter.

✨ Key Feature: Dual-Format Output

Each chapter is rendered in two formats:

  • HTML - For reading and documentation (you're viewing this now)
  • RevealJS - For presentations (click the easel icon 🎨)

How It Works

The book's _quarto.yml configuration specifies multiple output formats:

format: html: theme: cosmo revealjs: theme: night slide-number: true

Output Structure

When rendered, the book generates:

_book/ ├── index.html # HTML pages for reading ├── intro.html ├── chapter1.html ├── chapter2.html ├── summary.html └── revealjs/ # RevealJS presentations ├── index.html ├── intro.html ├── chapter1.html ├── chapter2.html └── summary.html
📝 Note: This is a mockup showing the UI enhancement. The actual implementation adds the easel icons dynamically based on the available output formats for each chapter, with proper links to the RevealJS presentations in the revealjs/ subdirectory.

Implementation Details

The feature is implemented through modifications to:

  • format-reveal.ts - Adds book extension support with multi-file output
  • book-config.ts - Generates revealjsHref for sidebar items
  • sidebaritem.ejs - Renders the RevealJS link icons
  • book.scss - Styles the alternate format links

Benefits

📚 Write Once, Present Anywhere

Maintain a single source of truth for your content that works for both documentation and presentations. The same Markdown files generate beautifully formatted HTML pages for reading and engaging RevealJS slide decks for presenting.
